CS239 Unix Systems Programming

2-Way Chat

Focus

  • Networking
  • Multi-plexing

Spec

Application: Allow two users to "chat"

  • We need a "server" that will sit there waiting for a client to connect.
  • After receiving a connection, both client and server will act alike.
  • Each will take whatever the user at their end types in and send it, together with the user's name to the other process.
  • They will also take whatever they other process sends over the network connection and display it to their user. Each transmission from one party to the other includes the sender's name. The receiver does not have to remember that name.
  • There "problem", aside from the usual setting up programs to act as TCP/IP client and server, is how does a process get input from two sources at a time?
  • One could constantly poll, with the downsides of wasted cpu usage and possibly additional delay in recognizing input due to sleeping.
  • One could assign a thread or even a process to the the task of reading from each file descriptor.
  • In this assignment you will use the Unix function select.

Interface

  • While it would be nice to have a pretty GUI for this program, we will do without one. (If we get to covering the Curses library it might be nice to add a GUI.)
  • Instead we will simply have the user type to the application at the shell and similarly see the output displayed to the shell.
  • To start either the client or the server, you will want to provide a name for the user on the command line.
  • The server should also accept a port number at the command line, so that we can run it flexibly using whatever port number is convenient.
  • The client will need to provide both the IP address of the server together with the port number that the server is listening on.

Turn in:

Submit your solution on Blackboard as a compressed tar file. It should include:

  • A makefile
  • The source file(s). It is possible to combine both client and server in one program. Feel free to do so if you like or write them as two separate programs. I have no preference.
  • README. Make sure it is clear how to run your program. Also, if there is anything I should know about, such as anything that doesn't work, explain it here.
