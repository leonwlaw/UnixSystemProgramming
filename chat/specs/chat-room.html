CS239 Unix Systems Programming

Chat Room

Focus

  • Concurrency
  • Networking

Spec

Write a chat room server

  • The chat server's primary job is to pass messages from all clients to all clients.
  • The server will need to
    • allow connections from clients.
    • receive messages
    • send the messages to all connected clients.

Notes

  • Each time a client connects a new thread should be created to handle the input from the client.
  • Each time a message is sent by a client, the message should be added to a queue.
  • A single separate thread should be responsible for removing the messages from the queue and sending them to the all of the clients.
  • Note the main areas to watch for race conditions:
    • Maintaining the "list" of clients currently logged in.
    • Maintaining the queue of messages to be sent out.
  • In order for the server to know that the client is "signing off", the client should send a message to the server saying so.

Turn in:

Submit your solution on Blackboard as a compressed tar file. It should include:

  • A makefile
  • The source files: Your server and a corresponding client. The latter might be the same client code that you used for the previous assignment.
  • README. Explain anything special about your program. Please note whether the client is the same as in the previous assignment, if not what changed. And, as always, explain if there is anything that didn't work.
